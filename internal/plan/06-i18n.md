# M6: i18n対応（UI / 出力言語の日英切替）

## 目的

UI と要約出力の両方で日本語 / 英語を切り替えられるようにする。
[`SPEC §3 国際化`](../../SPEC.md#3-非機能要件) と [`§5.2 サイドパネル`](../../SPEC.md#52-サイドパネル幅-ユーザー任意推奨400px) の要件を実装する。

## スコープ

- ✅ `_locales/{ja,en}/messages.json` の整備
- ✅ UI翻訳ヘルパ（`chrome.i18n` ベース＋手動切替の自前ロード層）
- ✅ オプションページに UI言語 / 要約出力言語の設定欄
- ✅ サイドパネルに出力言語のクイック切替ドロップダウン
- ✅ プロンプトテンプレート英語版
- ✅ YouTubeページ内ボタンのラベルもUI言語に追従

### 2. i18nヘルパ

`src/lib/i18n.ts`:

```ts
type LocaleStrings = Record<string, { message: string }>;
type Lang = 'ja' | 'en' | 'auto';

let cache: { lang: Lang; strings: LocaleStrings } | null = null;

const loadCustomLocale = async (lang: 'ja' | 'en'): Promise<LocaleStrings> => {
  const url = chrome.runtime.getURL(`_locales/${lang}/messages.json`);
  const res = await fetch(url);
  return await res.json();
};

export const initI18n = async (lang: Lang) => {
  if (lang === 'auto') {
    cache = null;
    return;
  }
  cache = { lang, strings: await loadCustomLocale(lang) };
};

export const t = (key: string): string => {
  if (cache) return cache.strings[key]?.message ?? key;
  return chrome.i18n.getMessage(key) || key;
};
```

### 3. 既存UIへの適用

サイドパネル / オプション / コンテンツスクリプトの全文字列を `t('xxx')` 経由に置換。

サイドパネル例:

```ts
import { initI18n, t } from '../lib/i18n';
import { loadSettings } from '../lib/storage';

const main = async () => {
  const settings = await loadSettings();
  await initI18n(settings.uiLanguage);
  document.querySelector('#title')!.textContent = t('extensionName');
  document.querySelector('#settings-btn')!.setAttribute('aria-label', t('settings'));
  // ...
};
```

YouTubeページ内ボタンの動的生成箇所:

```ts
const settings = await loadSettings();
await initI18n(settings.uiLanguage);
btn.innerHTML = `<span>🪄</span><span>${t('summarizeButton')}</span>`;
```

`sidepanel.html`:

```html
<div class="lang-selector">
  <label>${t('outputLanguage')}:
    <select id="output-lang-quick">
      <option value="ja">日本語</option>
      <option value="en">English</option>
    </select>
  </label>

```

切替時の挙動:
- 設定値（`outputLanguage`）は保存しない（あくまで一時切替）
- 「再生成」ボタンを押した時に新言語で再要約
- 単に値を変えただけでは再要約しない（API消費を防ぐ）

### 7. content scriptでの言語ロード

content script は `chrome.i18n.getMessage` がそのまま使えるが、手動切替には対応しないと統一感がない。
`mountActionButton()` の冒頭で `loadSettings()` → `initI18n()` を呼ぶ。
`chrome.storage.onChanged` を listenし、`uiLanguage` 変更時にボタンラベルを再描画する。

## 完了条件

1. オプションでUI言語を切り替えると、サイドパネル・オプション・YouTubeボタンのラベルが即座に追従する（リロード不要）
2. オプションで要約出力言語を切り替えると、次回の要約から新言語で出力される
3. サイドパネル上のクイック切替＋再生成で言語を変更できる
4. `auto` 設定時、ブラウザロケールが en なら英語、ja なら日本語UIになる

## 動作確認手順

1. UI言語 = Auto, ブラウザを英語ロケールに → 全UIが英語表示
2. UI言語 = 日本語に変更 → 即座に日本語表示に切替
3. 出力言語 = English にして要約 → Markdown が英語で生成される
4. サイドパネルのドロップダウンを 日本語 に切替 → 再生成ボタン押下 → 日本語で再生成
5. オプション保存後、ブラウザを再起動しても言語設定が維持される

## リスク / 注意点

- **`chrome.i18n` の制約**: ブラウザロケール固定で実行時切替不可なので、自前のフェッチ層が必要（`initI18n` で実装）
- **manifest中のローカライズ**: `manifest.json` の `name` / `description` も `__MSG_xxx__` で参照すれば各 `_locales/*/messages.json` から取得される
- **YouTube SPA との衝突**: ボタンラベル更新タイミング次第で一瞬旧言語が見える。`chrome.storage.onChanged` ハンドラで再挿入する
- **キャッシュ**: `i18n.ts` の `cache` は in-memory のみ。タブ間で同期が必要な場合は `chrome.storage.onChanged` で各コンテキストが個別にリロード
